# Figures

Rendered figures for the degree assessment. Figures 1–2 come from the D-SCRIPT
assessment thread; figures 3–7 come from the fully-reproducible companion
(`notebook/degree_bias_reproducible.ipynb`).

| file | caption |
|---|---|
| `fig1_exp_vs_pred_degree.png` | **(a)** Spearman corr(score, node degree): experimental scores (BioPlex 293T/HCT116) are negative (−0.10, −0.11 → specific low-degree top pairs); D-SCRIPT is strongly positive (+0.66 → hub-enriched top pairs). **(b)** Benchmark positives (mean degree 6.6) are hubs, negatives (mean 2.8) are not → a degree-only baseline scores AUROC 0.91. |
| `fig2_degree_leakage_yprob.png` | **(a)** A held-out link predictor using only node degree reaches AUC 0.70. **(b)** Re-evaluated on degree-preserving edge-swap scrambles, the AUC barely moves — the skill is degree, not biology. |
| `fig3_degree_is_confound.png` | **(a)** The connectivity "score" is essentially raw degree (ρ 0.94). **(b)** Across 2,000 degree-preserving scrambles, ACTB (hub) stays top-3 100% of the time; MCM6 (true member) stays at the bottom. |
| `fig4_hub_vs_member_network.png` | Network view of 3 curated complexes + hubs. ACTB (degree 23) edges into all three complexes and ranks #1 despite being a member of none; true member MCM6 (degree 6) ranks last. |
| `fig5_rank_vs_membership.png` | Connectivity leaderboard: hubs (ACTB, HSPA8, HSP90AA1) fill the top; every true MCM member sits at the bottom. Rank measures connectivity, not membership. |
| `fig6_specificity_recovers_signal.png` | **(left)** Degree vs specificity separates true members (low degree, high specificity) from promiscuous artifacts (high degree, low specificity). **(right)** For MCM6, evidence quality (max MI-score) — not partner count — flags real replication-machinery partners. |
| `fig7_three_step_filter.png` | Recovering targets: naive connectivity ranks the hub #1; a specificity × evidence × membership filter demotes ACTB/HSPA8 to "interactor, not member" and retains every true subunit as a confirmed target. |

All figures are regenerated by the notebook (fig3–7) or the D-SCRIPT assessment
pipeline (fig1–2). No manual editing.
